# AgroCRM — тезисы для защиты проекта

## 1. Коротко о проекте

**AgroCRM** — это CRM-система для фермерского хозяйства, которая помогает вести полный цикл управления: от полей и посевов до склада, урожая, продаж, прибыли и внешних заявок.

Главная идея: фермеру не нужно вести всё в тетрадях, Excel и мессенджерах — все ключевые процессы собраны в одной простой системе.

---

## 2. Какую проблему решаем

У небольших фермерских хозяйств часто нет удобного инструмента для контроля:

- сколько земли занято;
- сколько семян осталось;
- какие культуры посажены;
- какие задачи нужно выполнить;
- сколько урожая собрано;
- сколько продукции на складе;
- кому и сколько продали;
- какая прибыль по сезону;
- как принимать заявки от клиентов и новых фермеров.

AgroCRM закрывает этот цикл в одном MVP.

---

## 3. Главные фичи

### 3.1 Авторизация и заявки на доступ

- Есть экран входа.
- Есть форма заявки на подключение фермерского хозяйства.
- Заявки сохраняются в базе.
- Заявки отправляются на почту через Gmail SMTP.
- Добавлена отдельная админка для обработки заявок.

**Админ-доступ:**

```text
Логин: admin
Пароль: admin
```

---

### 3.2 Отдельная админ-панель

Сделана простая административная панель для AgroCRM.

Возможности:

- просмотр заявок на доступ;
- просмотр лидов с сайта;
- просмотр публичных заказов;
- счётчики новых обращений;
- смена статусов заявок:
  - новая;
  - в работе;
  - одобрена;
  - готово;
  - отклонена.

Админка сделана в общей цветовой палитре AgroCRM.

---

### 3.3 Управление фермой

- Создание и редактирование фермы.
- Профиль пользователя.
- Настройки хозяйства.
- Регион, адрес, данные владельца.

---

### 3.4 Управление полями

- Список полей.
- Площадь в гектарах.
- Локация.
- Тип почвы.
- Статус поля.
- Добавление, редактирование и удаление полей.
- Фильтрация и поиск.

---

### 3.5 Посевы

Одна из главных фич демо.

Реализован сценарий:

```text
поле → культура → посадка картофеля → списание семян со склада
```

Что есть:

- создание посева;
- выбор поля;
- выбор культуры;
- указание количества семян;
- автоматическое списание семян со склада;
- расчёт остатка семян;
- создание связанных задач;
- редактирование посева;
- завершение посева;
- удаление посева с возвратом семян.

---

### 3.6 Склад

- Учёт семян.
- Учёт урожая.
- Учёт прочих товаров.
- Остатки в килограммах.
- Минимальные остатки.
- Средняя себестоимость.
- Корректировка остатков.
- Предупреждения о низком запасе.

Пример для демо:

```text
Было 700 кг семян картофеля
Создали посев на 500 кг
Осталось 200 кг
```

---

### 3.7 Задачи

- Список задач по ферме.
- Связь задачи с полем и посевом.
- Тип задачи: посадка, полив, обработка, сбор урожая и т.д.
- Статусы задач.
- Отметка задачи как выполненной.
- Автоматическое создание задач при создании посева.

---

### 3.8 Урожай

- Добавление записи о сборе урожая.
- Привязка к посеву, полю и культуре.
- Количество урожая в кг.
- Качество урожая.
- Автоматическое добавление урожая на склад.
- Удаление урожая с обратным списанием со склада.

---

### 3.9 Клиенты

- Список клиентов.
- Тип клиента: ресторан, магазин, опт и т.д.
- Контакты.
- Адрес.
- Заметки.
- CRUD-операции.

---

### 3.10 Продажи

- Создание продажи.
- Выбор клиента.
- Выбор товара со склада.
- Количество в кг.
- Цена за кг.
- Автоматический расчёт:
  - выручки;
  - себестоимости;
  - прибыли.
- Автоматическое списание товара со склада.
- Удаление продажи возвращает товар на склад.

---

### 3.11 Dashboard и аналитика

На главном экране видно:

- выручку;
- прибыль;
- остаток урожая;
- остаток семян;
- ближайшие задачи;
- предупреждения;
- активные посевы;
- продажи;
- погоду по региону фермы.

Дополнительно есть аналитика:

- прибыль по культурам;
- сводка по складу;
- ближайшие задачи.

---

### 3.12 Публичное API

Реализованы endpoint'ы для внешних интеграций и демонстрации жюри:

```http
GET  /public/products
POST /public/orders
POST /public/leads
POST /public/applications
```

Что это даёт:

- внешний сайт/магазин может получить список продукции;
- клиент может оформить заказ;
- потенциальный клиент может оставить лид;
- фермер может отправить заявку на доступ.

---

### 3.13 Swagger-документация

Добавлена OpenAPI/Swagger-документация для backend API.

URL:

```text
http://localhost:8080/swagger
```

Документация включает:

- публичные endpoint'ы;
- CRUD по основным сущностям;
- admin endpoint'ы;
- схемы request/response;
- описание моделей.

---

### 3.14 Импорт и экспорт XLSX

Добавлены операции с Excel-файлами:

- экспорт данных в XLSX;
- импорт данных из XLSX.

Поддерживаются разделы:

- поля;
- склад;
- посевы;
- клиенты;
- задачи;
- урожай;
- продажи;
- аналитика.

---

### 3.15 Email-уведомления

Заявки и обращения отправляются на почту.

Поддерживается отправка для:

- заявки на доступ;
- лида;
- публичного заказа.

В ответе API есть поле:

```json
{
  "emailSent": true
}
```

## 4. Демо-сценарий для защиты

Рекомендуемый порядок показа:

1. Открыть AgroCRM.
2. Показать экран входа и заявку на доступ.
3. Войти в админку:

   ```text
   admin / admin
   ```

4. Показать, что заявки появляются в админке и можно менять статус.
5. Войти в фермерскую CRM.
6. Показать Dashboard.
7. Открыть склад и показать семена картофеля.
8. Создать посев картофеля на 500 кг семян.
9. Показать, что семена списались со склада.
10. Показать автоматически созданные задачи.
11. Добавить урожай.
12. Показать, что урожай появился на складе.
13. Создать клиента.
14. Создать продажу.
15. Показать прибыль на Dashboard/Analytics.
16. Открыть Swagger и публичное API.

## 7. Чем проект ценен для хакатона

Проект показывает не просто красивый интерфейс, а работающий end-to-end бизнес-процесс:

```text
заявка → админка → ферма → поле → посев → склад → задачи → урожай → продажа → прибыль → публичное API
```

Это MVP, который можно развивать дальше в полноценную платформу для фермерских хозяйств.
